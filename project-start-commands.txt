================================================================
 SYNTHETIC DATA GENERATION — LOCAL RUN COMMANDS
 Monorepo: web_api/ and workers/ are SEPARATE uv projects.
 Run BOTH services from this repo root (imports are rooted here).
================================================================

----------------------------------------------------------------
 0. ONE-TIME — install each service's deps
----------------------------------------------------------------
cd web_api ; uv sync ; cd ..
cd workers ; uv sync ; cd ..

----------------------------------------------------------------
 1. START SHARED INFRASTRUCTURE (Docker)
----------------------------------------------------------------
docker compose up -d            # redis, qdrant, minio, flower

# MongoDB runs separately (compose entry is commented out):
docker run -d --name synthetic_data_mongo -p 27017:27017 -v mongo_data:/data/db mongo:7

# every time after:
docker compose start
docker start synthetic_data_mongo

----------------------------------------------------------------
 2. RUN THE SERVICES (two terminals, BOTH from repo root)
----------------------------------------------------------------
# Terminal 1 — web_api  (http://localhost:8000)
web_api\.venv\Scripts\activate
uvicorn web_api.main:app --reload --port 8000

# Terminal 2 — workers  (--pool=solo REQUIRED on Windows)
workers\.venv\Scripts\activate
celery -A workers.celery_app worker --loglevel=info --pool=solo

# Per-service detail: web_api/README.md and workers/README.md

----------------------------------------------------------------
 3. .ENV (shared, at repo root — both services read it)
----------------------------------------------------------------
MONGODB_URL, DATABASE_NAME, MINIO_*, ENCRYPTION_KEY,
OPENAI_API_KEY, GEMINI_API_KEY, (opt) CELERY_*, QDRANT_URL, WEB_API_URL

Generate ENCRYPTION_KEY:
  web_api\.venv\Scripts\python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

----------------------------------------------------------------
 4. VERIFY
----------------------------------------------------------------
docker ps
API:    http://localhost:8000/docs
Qdrant: http://localhost:6333/dashboard
MinIO:  http://localhost:9001  (minioadmin / minioadmin)
Flower: http://localhost:5555

----------------------------------------------------------------
 5. STOP / CLEANUP
----------------------------------------------------------------
# Ctrl+C the two service terminals
docker compose stop                     # stop infra (keep data)
docker stop synthetic_data_mongo
docker compose down -v                  # WIPE infra volumes

----------------------------------------------------------------
 6. COMMON ISSUES
----------------------------------------------------------------
- Celery hangs on tasks            -> forgot --pool=solo (Windows)
- ModuleNotFoundError web_api/...  -> not run from repo root, or wrong venv active
- `import torch` works in web_api  -> bad split; torch belongs ONLY to workers
- Connection refused 6379/6333     -> infra container not started
- Credentials endpoints 500        -> ENCRYPTION_KEY missing
- First academic PDF very slow     -> Marker downloads models once
